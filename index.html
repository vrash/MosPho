MosPho

Basic image mosaic creator. Also Phil Karlton apparently once said

There are only two hard things in Computer Science: cache invalidation and naming things

Hence the poor name choice. The 's' wasn't there in the original title.

Implementation

Bare bones. As much background processing as possible. No Picasso/Fresco/Retrofit and other things. No real OOP. Does the job for an MVP though, can be put into a full blow app as a module to ship. If there's no network, I pick up average chunk colours and display the mosaic. Url - Bitmaps are cached in a LRU bitmap cache, so it should get faster on more use. No disk cache for now. No tests for now, but stubs can easily be put in for critical pieces and a latch based test mechanism for the async tasks. Overall core logic time taken -- About 3.5 - 4 hours. Commit History gives a rudimentary sense of progress

Logic

Very Simple.

  • FETCH IMAGE FROM WHEREVER THE USER WANTS TO
  • SPLIT IMAGE INTO SMALLER PIECES
  • RUN EACH ROW THROUGH THE NETWORK TO FIND THE EQUIVALENT IMAGE (OR CACHE)
  • STITCH THE IMAGE BACK (TODO: ROW BY ROW STITCHING)
  • DISPLAY MOSAIC AND VOILA, WE'RE DONE.

MinSDK

KitKat - API 19. Reason is that still a large % of users are on kitkat and although tempting to set minSDK to Lollipop (and get all the Palette class goodness) this is better for bare bones coding and manipulation.
